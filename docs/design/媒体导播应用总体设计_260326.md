# pymediamixer 媒体导播应用 -- 总体设计方案

---

## 1. 总体架构

三段流水线（Pipeline）架构，段间通过 `gst-plugin-rs` 的 `rsinter` 插件中的 `intersink` / `intersrc` 进行解耦连接：

```
+------------------+        intersink/intersrc        +---------------------+        intersink/intersrc        +-------------------+
|  Input Pipeline  | ===============================> | Compositing Pipeline | ===============================> |  Output Pipeline  |
|  (per source)    |  producer-name: input0_video     |  (per media type)   |  producer-name: comp0_video     |  (per destination) |
+------------------+  producer-name: input0_audio     +---------------------+                                  +-------------------+
|  source element  |                                  |  compositor/mixer   |                                  |  autovideosink    |
|  -> videoconvert |                                  |  N路 intersrc       |                                  |  / rtmpsink 等    |
|  -> intersink    |                                  |  -> 合成处理        |                                  +-------------------+
|  -> audioconvert |                                  |  -> intersink       |
|  -> intersink    |                                  +---------------------+
+------------------+
```

### intersink / intersrc 核心特性（来自 rsinter 插件）

| 特性 | 说明 |
|---|---|
| 元素名称 | `intersink`（sink端，生产者） / `intersrc`（source端，消费者） |
| 关键属性 | `producer-name`（字符串），通过匹配此名称连接 sink 与 src |
| 运行时切换 | `producer-name` 在两端都是 `mutable_playing`，可在 PLAYING 状态下动态切换 |
| 1:N 支持 | 一个 intersink（producer）可被多个 intersrc（consumer）消费 |
| Caps 无关 | ANY caps，可传输任意媒体类型（视频/音频/字幕等） |
| 内部实现 | 基于 StreamProducer API，内部封装 appsink（intersink）和 appsrc（intersrc） |
| 事件转发 | `event-types` 属性控制转发的事件类型（默认转发 EOS） |
| 队列控制 | intersrc 支持 `max-buffers` / `max-bytes` / `max-time` 限制 |
| 同步 | intersink 的 `sync` 属性控制是否同步时钟（默认 true） |

### 关键设计原则

- **错误隔离**：每个 Pipeline 是独立的 `Gst.Pipeline`，拥有独立的 GLib.MainContext / GLib.MainLoop / 线程。一段的错误/EOS 不影响其他段。
- **动态切换**：通过修改 `intersrc` 的 `producer-name` 属性，运行时切换到不同的输入源。
- **无启动顺序限制**：输入、合成、输出管线可任意次序创建/启动/连接。intersrc 在对端 intersink 未就绪时不会崩溃。
- **全局时钟同步**：所有管线共享全局 `SystemClock` 和全局 `base_time`，确保时间戳一致。

## 2. 线程模型与时钟同步

### 2.1 独立线程

每个 Pipeline 独立运行在自己的线程中，拥有独立的 GLib.MainContext 和 GLib.MainLoop：

```python
class PipelineBase:
    def __init__(self, name):
        self._context = GLib.MainContext.new()       # 独立的 GLib 上下文
        self._mainloop = GLib.MainLoop.new(self._context, False)
        self._thread = None                          # 后台运行线程

    def start(self):
        """非阻塞启动：构建 pipeline，设置时钟，启动后台线程"""
        self._build()
        self._apply_clock()       # 应用全局 SystemClock + base_time
        self._pipeline.set_state(Gst.State.PLAYING)
        self._thread = threading.Thread(target=self._run, daemon=True)
        self._thread.start()

    def _run(self):
        """线程入口：推入线程默认上下文，添加 bus watch，运行 mainloop"""
        self._context.push_thread_default()
        bus = self._pipeline.get_bus()
        bus.add_signal_watch()
        bus.connect("message::error", self._on_error)
        bus.connect("message::eos", self._on_eos)
        bus.connect("message::state-changed", self._on_state_changed)
        bus.connect("message::latency", self._on_latency)
        self._mainloop.run()
        self._context.pop_thread_default()

    def stop(self):
        """停止 pipeline，退出 mainloop，等待线程结束"""
        self._pipeline.set_state(Gst.State.NULL)
        self._mainloop.quit()
        if self._thread:
            self._thread.join(timeout=5)
```

### 2.2 全局时钟同步

```python
# core/clock.py -- 模块级全局时钟

_global_clock = Gst.SystemClock.obtain()
_global_base_time = _global_clock.get_time()

def apply_clock(pipeline: Gst.Pipeline):
    """将全局时钟和 base_time 应用到指定 pipeline"""
    pipeline.set_clock(_global_clock)
    pipeline.set_start_time(Gst.CLOCK_TIME_NONE)
    pipeline.set_base_time(_global_base_time)
```

每个 Pipeline 在 `start()` 中自动调用 `apply_clock()`。无需手动管理，无启动顺序依赖。

### 4.2 PipelineBase（所有流水线的基类）

```
src/pymediamixer/core/pipeline_base.py
```

**职责**：生命周期管理、独立线程运行、bus 消息处理、全局时钟、状态查询、**自动重启**、日志。

| 方法 | 说明 |
|---|---|
| `__init__(name: str, auto_restart=True)` | 初始化，创建 MainContext/MainLoop；`auto_restart` 控制 ERROR/EOS 后是否自动重建管线 |
| `start()` | 非阻塞启动：调用 `_build()`，应用全局时钟，设 PLAYING，启动线程 |
| `stop()` | 停止 pipeline，退出 mainloop，join 线程 |
| `restart()` | 销毁当前 pipeline，重新调用 `_build()` 构建全新管线，重新启动 |
| `get_state() -> str` | 返回当前状态 |
| `wait(timeout=None)` | 阻塞等待 pipeline 停止（EOS 或错误） |
| `_build()` | **抽象方法**，子类实现 |
| `_on_error(bus, msg)` | 错误处理：记录日志，回调通知；若 `auto_restart=True` 则自动调用 `restart()` |
| `_on_eos(bus, msg)` | EOS 处理：记录日志，回调通知；若 `auto_restart=True` 则自动调用 `restart()` |
| `_on_latency(bus, msg)` | 调用 `pipeline.recalculate_latency()` |

**自动重启机制**：

当管线收到 ERROR 或 EOS 消息时，若 `auto_restart=True`（默认），PipelineBase 会自动执行以下步骤：
1. 将当前 pipeline 设为 NULL 状态（释放所有资源）
2. 销毁旧的 Gst.Pipeline 对象
3. 重新调用 `_build()` 构建全新的 GStreamer pipeline（包括所有 element 和 link）
4. 重新应用全局时钟
5. 设为 PLAYING 状态恢复运行

重启在 pipeline 自己的线程中异步执行，不阻塞调用方。重启间设置短暂延迟（如 1 秒）避免快速循环重启。连续重启次数超过阈值（如 5 次/分钟）时停止自动重启，仅记录日志并回调通知。

**状态回调**：`on_state_changed`、`on_error`、`on_eos`、`on_restarted` 回调属性，允许外部注册监听。

### 4.3 InputPipeline（输入流水线基类）

```
src/pymediamixer/core/input_pipeline.py
```

**继承**：`PipelineBase`

**职责**：管理输入源到 1~2 个 `intersink` 的数据流。一个输入管线可同时输出视频和音频（各一个 intersink），也可仅输出一种。同一种媒体类型最多一个 intersink。

| 方法/属性 | 说明 |
|---|---|
| `__init__(name, media_types: list[MediaType])` | 初始化，指定输出端媒体类型列表，如 `[MediaType.VIDEO]` 或 `[MediaType.VIDEO, MediaType.AUDIO]` |
| `get_channel(media_type: MediaType) -> str` | 返回指定媒体类型的 producer-name，格式：`{name}_{media_type.value}` |
| `channels -> dict[MediaType, str]` | 只读属性，返回所有输出通道 `{MediaType: producer_name}` |
| `_build()` | 模板方法：调用 `_create_source()` -> `_create_processing()` -> 创建 intersink |
| `_create_source() -> Gst.Element 或 dict` | **抽象方法**，子类实现。返回单个 element 或按媒体类型返回 element 字典 |
| `_create_processing(media_type) -> list[Gst.Element]` | 可选覆盖，返回中间处理 element 列表 |

**producer-name 命名规范**：`{pipeline_name}_{media_type}`，例如 `input0_video`、`input0_audio`。

**典型 GStreamer pipeline 结构**（仅视频输出）：
```
source -> videoconvert -> capsfilter -> intersink(producer-name="input0_video")
```

**典型结构**（视频+音频输出，例如文件/RTSP输入）：
```
demux -> videoconvert -> capsfilter -> intersink(producer-name="input0_video")
      -> audioconvert -> capsfilter -> intersink(producer-name="input0_audio")
```

### 4.4 CompositingPipeline（合成流水线基类）

```
src/pymediamixer/core/compositing_pipeline.py
```

**继承**：`PipelineBase`

**职责**：一条合成管线仅处理**一种**媒体类型。拥有多个 channel（每个 channel 对应一个 intersrc 输入），可动态切换每个 channel 连接的输入源。合成后通过 intersink 输出。

| 方法/属性 | 说明 |
|---|---|
| `__init__(name, media_type: MediaType, num_channels: int, output_caps=None)` | 初始化，指定媒体类型、channel 数量、输出参数 |
| `media_type -> MediaType` | 只读属性 |
| `output_channel -> str` | 输出 producer-name：`{name}_{media_type.value}` |
| `num_channels -> int` | channel 数量 |
| `switch_channel(channel_index: int, input_pipeline: InputPipeline)` | **核心方法**：将指定 channel 切换到某个输入管线。自动根据 `self.media_type` 选择该输入管线对应媒体类型的输出端 |
| `switch_channel_by_name(channel_index: int, producer_name: str)` | 直接通过 producer-name 切换 |
| `get_channel_source(channel_index: int) -> str` | 返回指定 channel 当前连接的 producer-name |
| `set_channel_layout(channel_index: int, layout: dict)` | 设置 channel 的布局参数（视频合成专用） |
| `_build()` | 模板方法：创建 N 个 intersrc -> 处理链 -> compositor -> intersink |
| `_create_compositor() -> Gst.Element` | **抽象方法**，子类实现 |
| `_apply_layout(pad, layout: dict)` | 将布局参数应用到 compositor sink pad（子类实现） |

**动态切换机制**（核心）：
```python
def switch_channel(self, channel_index: int, input_pipeline: InputPipeline):
    """将指定 channel 切换到某个输入管线"""
    producer_name = input_pipeline.get_channel(self._media_type)
    self._intersrcs[channel_index].set_property("producer-name", producer_name)

def switch_channel_by_name(self, channel_index: int, producer_name: str):
    """直接通过 producer-name 切换"""
    self._intersrcs[channel_index].set_property("producer-name", producer_name)
```

**channel 布局配置**（视频合成时每个 channel 的位置参数）：
```python
{
    "xpos": 0,
    "ypos": 0,
    "width": 1920,       # 缩放到目标尺寸
    "height": 1080,
    "zorder": 0,         # 层次，越大越前
    "alpha": 1.0         # 透明度
}
```

**典型 GStreamer pipeline 结构**（视频合成，2 channel）：
```
intersrc(producer-name=?) -> videoconvert -> videoscale -> capsfilter(WxH) -> compositor.sink_0
intersrc(producer-name=?) -> videoconvert -> videoscale -> capsfilter(WxH) -> compositor.sink_1
                                                            compositor -> videoconvert -> capsfilter -> intersink(producer-name="comp0_video")
```

```python
class VideoCompositor(CompositingPipeline):
    """基于 compositor 的视频合成器"""
    def __init__(self, name, num_channels=2,
                 width=DEFAULT_VIDEO_WIDTH, height=DEFAULT_VIDEO_HEIGHT,
                 framerate=DEFAULT_VIDEO_FRAMERATE):
        output_caps = {"width": width, "height": height, "framerate": framerate}
        super().__init__(name, media_type=MediaType.VIDEO,
                         num_channels=num_channels, output_caps=output_caps)

    def _create_compositor(self):
        comp = Gst.ElementFactory.make("compositor", f"{self._name}_compositor")
        comp.set_property("background", 1)  # black background
        return comp

    def _apply_layout(self, pad, layout):
        pad.set_property("xpos", layout.get("xpos", 0))
        pad.set_property("ypos", layout.get("ypos", 0))
        pad.set_property("width", layout.get("width", 0))
        pad.set_property("height", layout.get("height", 0))
        pad.set_property("zorder", layout.get("zorder", 0))
        pad.set_property("alpha", layout.get("alpha", 1.0))
```

## 6. MixerEngine 编排引擎

```
src/pymediamixer/engine.py
```

**职责**：统一管理所有 Pipeline 的创建、连接、生命周期。这是一个可选的便捷封装，用户也可以直接使用三种管线类自行管理。

```python
class MixerEngine:
    def __init__(self):
        self._inputs: dict[str, InputPipeline] = {}
        self._compositors: dict[str, CompositingPipeline] = {}
        self._outputs: dict[str, OutputPipeline] = {}

    # --- Pipeline 管理 ---
    def add_input(self, input_pipeline: InputPipeline)
    def add_compositor(self, comp_pipeline: CompositingPipeline)
    def add_output(self, output_pipeline: OutputPipeline)
    def remove_input(self, name: str)
    def remove_output(self, name: str)

    # --- 连接与切换 ---
    def switch(self, compositor_name: str, channel_index: int, input_name: str)

    # --- 生命周期（无顺序限制，各管线可独立启停）---
    def start_all(self)
    def stop_all(self)
    def start_input(self, name: str)
    def start_compositor(self, name: str)
    def start_output(self, name: str)
    def get_status() -> dict

    # --- 便捷工厂 ---
    @classmethod
    def from_config(cls, config: dict) -> "MixerEngine"
```

### SDK 使用示例

```python
from pymediamixer.inputs.videotestsrc_input import VideoTestSrcInput
from pymediamixer.compositors.video_compositor import VideoCompositor
from pymediamixer.outputs.autovideosink_output import AutoVideoSinkOutput
from pymediamixer.engine import MixerEngine

engine = MixerEngine()

# 添加两路输入
input0 = VideoTestSrcInput("input0", pattern="smpte", width=1920, height=1080)
input1 = VideoTestSrcInput("input1", pattern="ball", width=640, height=480)
engine.add_input(input0)
engine.add_input(input1)

# 添加视频合成器（2 channel），配置画中画布局
comp = VideoCompositor("comp0", num_channels=2, width=1920, height=1080)
engine.add_compositor(comp)

# 添加输出
output = AutoVideoSinkOutput("preview", source_producer_name="comp0_video")
engine.add_output(output)

# 启动所有管线（无顺序限制）
engine.start_all()

# 连接：channel 0 显示 input0，channel 1 显示 input1
engine.switch("comp0", 0, "input0")
engine.switch("comp0", 1, "input1")

# 设置布局：channel 0 全屏背景，channel 1 右下角画中画
comp.set_channel_layout(0, {"xpos": 0, "ypos": 0, "width": 1920, "height": 1080, "zorder": 0})
comp.set_channel_layout(1, {"xpos": 1280, "ypos": 600, "width": 640, "height": 480, "zorder": 1})

# 动态切换：将 channel 0 切换到 input1
engine.switch("comp0", 0, "input1")

engine.stop_all()
```

## 9. 错误隔离

| 场景 | 行为 |
|---|---|
| Input Pipeline ERROR | 记录日志，回调通知，**自动重建整条管线并重启**。intersrc 无新数据期间，compositor 保持最后状态 |
| Input Pipeline EOS | 记录日志，回调通知，**自动重建整条管线并重启**。intersink 已配置不转发 EOS，compositor 不受影响 |
| Output Pipeline ERROR | 记录日志，回调通知，**自动重建整条管线并重启**。合成管线不受影响 |
| Compositing Pipeline ERROR | 严重错误，记录日志，回调通知，**自动重建整条管线并重启** |

所有管线均通过 PipelineBase 的 `auto_restart` 机制（默认开启）在 ERROR/EOS 后自动重建整条管线并重启。

**intersink 的 `event-types` 配置（已确定）**

输入管线的 intersink **强制设置 `event-types` 为空列表**，不转发任何事件（含 EOS）：
```python
intersink.set_property("event-types", Gst.ValueArray([]))  # 不转发任何事件
```
这样即使输入源 EOS 或重启，合成管线也不会受到影响。此设置在 InputPipeline 基类的 `_build()` 中统一处理。

## 12. 待讨论的设计细节

1. **compositor pad 的 width/height**：GStreamer `compositor` sink pad 直接支持 `width`/`height` 属性缩放输入。替代方案是在 intersrc 之后用 `videoscale` + `capsfilter` 显式缩放，兼容性更好。建议 V1 先使用 compositor pad 属性（更简洁），如遇兼容性问题再改为显式缩放。

2. **日志方案**：Python `logging` 模块，每个 Pipeline 独立 logger（`pymediamixer.inputs.input0`），按模块过滤。

3. **intersrc 缓冲策略**：intersrc 的 `max-time` 默认 500ms。对于低延迟场景可能需要调小，可在基类中提供配置入口。

## 13. 已确认的设计决策

| 决策项 | 结论 |
|---|---|
| intersink event-types | 输入管线的 intersink 设置为空列表 `[]`，不转发任何事件 |
| 输入侧 caps 策略 | 输入管线保持原始分辨率，仅做格式转换（如 videoconvert），不做缩放。缩放由 compositor 侧负责 |
| 错误/EOS 处理 | 所有管线默认开启 `auto_restart`，自动重建整条管线并重启 |
| gst-plugin-rs | 已在 conda 环境中安装，源码仅供参考 |
| CLI 脚本位置 | 全局 CLI 放 `scripts/` 目录；单元测试 CLI 放 `test/` 目录 |
| 启动顺序 | 无限制，任意顺序创建/启动/连接 |
